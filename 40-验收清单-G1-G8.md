## G3 · CI 时 check 是否同步

| 判定 | 方法 | 对应文档 |
|---|---|---|
| `smarty check --ci --json` 输出含 `ok/missing/stale/drift/outOfDepth/exitCode` 5+1 字段 | jq | [18-step9](./18-step9-check-CLI-深层依赖.md) |
| 源文件改 1 行（含深度 5 内传递依赖） → `stale` 数组非空，exitCode=1 | shell | [18-step9](./18-step9-check-CLI-深层依赖.md) |
| 改 `design-tokens.ts`（在 ignoreGlobs） → 不触发 STALE | shell | [18-step9 §3](./18-step9-check-CLI-深层依赖.md) |
| 改 `<Bound deps>` → `drift` 非空 | shell | [18-step9 §3.4](./18-step9-check-CLI-深层依赖.md) |
| `git clone` 后直接跑 check：mtime 全新但内容未变 → up to date | CI fresh clone | [18-step9 D3](./18-step9-check-CLI-深层依赖.md) |
| `check --staged` 100 staged 文件 < 1s | benchmark | [18-step9 §3.6](./18-step9-check-CLI-深层依赖.md) |

| GitHub Actions PR 注释含 MISSING/STALE/DRIFT 三段 | workflow run | [19-step10 §3.2](./19-step10-pre-commit-与-CI.md) |
